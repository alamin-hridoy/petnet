Introduction

The guide will walk you through to tryout the services provided by Connex using the public API available on the platform. The documentation of the API is available in this link. Connex API is using OAuth2.0 and OpenID Connect and can available on below environment

Environment Explanation
Sandbox Get started with test credentials and life-like data
Production Ready to transact with real customer

Getting Started

Simple steps to use Connex API

  1. 1. Explore the API documentation portal
  2. 2. Download API definition
  3. 3. Generate API access key
    Connex platform support two methods of authorization as per OAuth2.0 flow and
    OpenID Connect.
    User can choose from two method below
    1. 1. Generate client credential
      1. 1. Login to Connex with your registered email address
      2. 2. Access the API Integration
      3. 3. Select API Keys
      4. 4. Select sandbox or production (available only upon approval)
      5. 5. Click on “Generate Client ID & Secret”
      6. 6. Enter your password
      7. 7. Connex will generate clientID and clientSecret
        1. 1.Do not share the clientID, clientSecret to anyone else outside your organization
        2. 2. The clientSecret is will be displayed only once, make sure to copy and keep it safe.
      8. 8. Use the clientID, clientSecret in OAuth2.0 flow to access Connex API
    2. 2. Generate authorization code
      1. 1. Login to Connex with your registered email address
      2. 2. Access the API Integration
      3. 3. Select OAuth2.0 Authorization Code
      4. 4. Select sandbox or production (available only upon approval)
      5. 5. Fill the required data
        1. 1. Name
        2. 2. Logout URL
        3. 3. Logo URL
        4. 4. Cors Origin
        5. 5. Redirect URL
        6. 6. Permission: Public / Private
        7. 7. Scope
      6. 6. Click on “Generate Credentials”
      7. 7. Enter your password
      8. 8. Connex will generate clientID and clientSecret
        1. 1. Do not share the clientID, clientSecret to anyone else outside your organization
        2. 2. The clientSecret is will be displayed only once, make sure to copy and keep it safe.
      9. 9. Use the clientID, clientSecret in OAuth2.0 flow to access Connex API
  4. 4. Test using preferred testing tools You can use the your preferred testing tools that support OAuth2.0 and OpenID Connect. The latest API Definition will be always available in the documentation portal for you to import into your testing tools
  5. 5. Request approval to access production environment
